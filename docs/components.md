# {{ $frontmatter.title }}

TDS는 **Toss Design System**의 약자예요.\
토스 커뮤니티 전반에서 사용하는 디자인 시스템으로, 토스 제품을 구성하는 공통의 디자인 언어이자 개발자의 도구예요.\
TDS는 디자이너, 개발자, 기획자 등 직군을 구분하지 않고 같은 기준으로 협업할 수 있게 도와줘요.

:::tip TDS 장점

* 사용자에게 일관된 제품 경험을 제공할 수 있어요.
* 디자이너는 UI 구현보다 문제 해결에 집중할 수 있어요.
* 개발자는 커스텀 UI를 만드는 것보다3~5배 빠르게 개발할 수 있어요.
  :::

## TDS 사용 시 유의사항

TDS를 안전하고 올바르게 사용하려면 아래 내용을 먼저 확인해 주세요.

:::tip 라이선스 안내
본 UI Kit을 사용함으로써, 본 UI Kit에 포함된 [라이선스](/design/prepare/figma-ui-license.html)의 조건에 동의하는 것으로 간주됩니다.
:::

### 1. 지식재산권

토스가 앱인토스(Appintoss) 서비스를 통해 제공하는 모든 자료의 권리는 지식재산권을 포함해 모두 토스에 귀속돼요. 파트너사는 해당 자료를 앱인토스 서비스 이용을 위한 범위 내에서만 사용할 수 있어요.

### 2. 사용 권한 범위

토스의 TDS 사용 허가는 앱인토스 서비스 제공을 위한 제한적인 사용 권한이에요. 이 범위를 넘어서는 결과물이나 추가적인 권리를 취득할 수는 없어요.

### 3. 준수 의무 및 위반 시 조치

파트너사는 본 가이드와 관련 법령을 반드시 준수해 주세요. 이를 위반할 경우, 토스는 앱인토스 서비스 제공을 중단하거나 기타 필요한 조치를 취할 수 있어요.

## 컴포넌트 리스트

앱인토스에서 가장 자주 사용하는 **핵심 TDS 컴포넌트 11개**를 소개해요.\
각 컴포넌트를 눌러 사용 방법과 활용 예시를 자세히 확인해 보세요.
