# 혜화동 안심경로 프로토타입

## 목적
스토킹 피해자 보호용 안심 길찾기 모바일 웹 + PWA. CCTV 촬영범위 안으로 경로를 유지해 사후 증거(영상 열람, 보관 30일) 확보 가능 구간을 최대화한다. 공모전 시연용 프로토타입. 서비스 범위는 종로구 혜화동 권역으로 한정.

## 아키텍처 원칙
- 백엔드/DB 없음. 모든 데이터는 빌드타임 정적 파일 (public/data/*.geojson, graph.json, stats.json)
- 라우팅은 브라우저에서 A* 직접 실행 (lib/astar.ts)
- 지도는 react-leaflet + OSM 타일. Leaflet 컴포넌트는 반드시 dynamic import + ssr:false
- 모바일 우선 (390px 기준). PWA: manifest + service worker로 홈 화면 설치 및 정적 데이터 오프라인 캐시

## 디자인
- 디자인의 최종 기준 문서는 docs/components.md. 모든 UI 컴포넌트는 이 문서의 토큰·컴포넌트 규칙을 따른다.
- 브랜드 그라데이션 #0083FF→#4C2CE2, iOS Liquid Glass 스타일 (반투명 블러 카드, 바텀시트)
- [디자인 요약] docs/components.md는 토스 디자인 시스템(TDS) 가이드 문서로, 앱인토스 미니앱 개발 시 사용하는 핵심 UI 컴포넌트 11종을 기준으로 삼는다. 디자이너·개발자·기획자가 동일한 기준으로 협업하도록 만든 공통 디자인 언어이므로, 커스텀 UI를 임의로 만들기보다 TDS 컴포넌트 규칙을 우선 재사용한다. 단, 해당 문서에는 구체적인 색상 hex 값·타이포 스케일 표는 포함되어 있지 않으므로, 브랜드 그라데이션(#0083FF→#4C2CE2)과 Liquid Glass 톤은 위 항목 기준을 따르고 세부 컴포넌트 스펙은 TDS 11종 컴포넌트 사용법을 참고해 확장한다. TDS 자료의 저작권은 토스에 귀속되며 본 프로토타입(앱인토스 서비스 범위) 내 용도로만 사용한다.

## 안전점수 공식 (전처리에서 사전 계산)
- CCTV 커버리지: 촬영방면 "360도 전방면"→반경 40m, "카메라 전면(고정)"→반경 30m, 카메라대수 2 이상이면 +10m
- 링크를 10m 간격 샘플점으로 나눠 coverage_ratio = 커버된 점 / 전체 점
- light_score = min(반경 25m 내 보안등 수 / 3, 1.0) — 보안등 1,806개 확보 완료이므로 항상 계산
- safety = 0.7 × coverage_ratio + 0.3 × light_score

## 경로 3종 비용함수
① 최단: length ② 균형: length×(1.5−safety) ③ 안심(추천): length×(2.5−2×safety)
②③ 결과가 같으면 ② 제거하고 2개만 표시.
